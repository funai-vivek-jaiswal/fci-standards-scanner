CRM · Rule CRM-004

Field API name violates naming convention

📁 Source Record: Account "Acme Corp" (ID: 3829481923) 📄 Standard: 04_Zoho_CRM_Standards.md §2 🕐 Detected: 27 May 2026 14:32
Finding Detail
Field with display name "Lead Source Detail" has API name lead_src which violates the UPPER_SNAKE_CASE convention. Expected API name: Lead_Source_Detail. API field names are permanent once created — this requires delete and recreate with data migration.
🔧 Remediation Steps
  1. 1 Go to CRM → Settings → Modules → Leads → Fields and locate the field with display name "Lead Source Detail"
  2. 2 Note: API field names cannot be changed in Zoho CRM — you must delete and recreate the field
  3. 3 Export all current values for this field using Data Export → Leads before deleting
  4. 4 Create a new field with API name Lead_Source_Detail and display name "Lead Source Detail"
  5. 5 Write a standalone function CRM_MigrateLeadSrc_standalone to copy old values to the new field
  6. 6 Delete the old lead_src field and re-run the scan to verify
